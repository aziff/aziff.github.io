Have you have been frustrated with disorganized .bib files? Have you lost a printed copy of a paper with all your annotations? Or worse yet, have you forgotten about a paper you wanted to read? Over the last year, I have been refining my **citation management system** to address these issues. In this post, I will describe my system and explain how I decided on each of its elements. 

<h3>
	Problems, Goals, and Solutions
</h3>

<h4>Citation Database</h4>
<ul>
<li> Problems: Unruly, incomplete, and erroneous .bib files. Bibliographies for different projects getting confused. </li>
<li style="color:#339898;">Goal: Create a citation database with options for categorization, tags, and notes. Seamless export to a BibTeX file is a must. </li>
<li> Solution: Zotero. <a href="https://guides.lib.uw.edu/c.php?g=344197&p=2318203">Comparing different management systems</a> and trying them out resulted in Zotero working the best for me. The desktop application has a nice interface for categorizing, editing, and adding references. The web application allows for collaboration, although this is a feature I have not explored yet. Importing references and exporting to a .bib file is simple. </li>
</ul>

<h4>PDF Organization and Notation</h4>
<ul>
<li> Problems: Lost or damaged printed copies. Even if printed copies are organized, portability is an issue. On an e-reader, the chosen application must have high-quality notation capabilities.</li>
<li style="color:#339898;">Goal: Create a consistent system for storing, reading, and annotating papers.</li>
<li>Solution: GoodReader. There are a <a href="https://thesweetsetup.com/apps/best-pdf-manager-editor-ipad/">myriad of applications to read PDFs on the iPad or other tablets.</a> I found that GoodReader has a smooth interface to notate and organize PDFs. The top competitor was Notability, but I do not usually take long-form notes on my iPad, so I did not feel the need to integrate PDF notation with an electronic notebook. </li>
</ul>

<h4>System Integration</h4>
<ul>
<li> Problems: Papers I read are not in my .bib files, or I see a citation but cannot find my notes on the paper. </li>
<li style="color:#339898;"> Goal: Harmonize citation database with paper organization. </li>
<li> Solution: Dropbox or any other cloud storage can be used to link Zotero and GoodReader. Mendeley has a more integrated way to store papers within the citation database, but the annotating capabilities are extremely limited. My solution allows for integration, although it requires some manual work to ensure that the .bib and my GoodReader folders are harmonized. </li>
</ul>

<h3>
	Putting It All Together
</h3>

When I encounter a paper I want to read, I follow these steps to ensure it is entered in my system.
<ol>
	<li>Download a PDF of the paper from Google Scholar. Change the name of the PDF to follow my naming convention: the last names of the authors separated by hyphens, an underscore, and the publication year. Store the PDF in my synced Dropbox folder, within the appropriate subfolder for the paper's topic.</li>
	<li>Drag and drop the PDF into Zotero. The meta data from the PDF allow for automated creation of an entry in the database. If applicable, I store the entry in a topic-specific subfolder of Zotero.</li>
	<li>My synced folder on GoodReader updates automatically, so I can start reading and annotating immediately.</li>
</ol>
